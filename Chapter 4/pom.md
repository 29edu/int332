# Project Object Model (POM)

The POM is the fundamental unit of Maven. It is an XML file named `pom.xml` placed at the root of every Maven project. It describes everything Maven needs to know about the project: its identity, dependencies, build configuration, and plugins.

---

## Minimal pom.xml

```xml
<project>
  <modelVersion>4.0.0</modelVersion>

  <groupId>com.example</groupId>
  <artifactId>my-app</artifactId>
  <version>1.0.0</version>
</project>
```

These three coordinates uniquely identify any Maven artifact:

- **groupId** — your organisation or domain, reversed (e.g. `com.example`, `org.springframework`)
- **artifactId** — the name of this specific project or module
- **version** — the version of this artifact

Together these are called the GAV coordinates.

---

## Full pom.xml Structure

```xml
<project xmlns="http://maven.apache.org/POM/4.0.0"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://maven.apache.org/POM/4.0.0
                             http://maven.apache.org/xsd/maven-4.0.0.xsd">

  <modelVersion>4.0.0</modelVersion>

  <!-- Project identity -->
  <groupId>com.example</groupId>
  <artifactId>my-app</artifactId>
  <version>1.0.0</version>
  <packaging>jar</packaging>

  <!-- Human-readable info -->
  <name>My Application</name>
  <description>A sample Maven project</description>

  <!-- Java version and encoding -->
  <properties>
    <java.version>17</java.version>
    <maven.compiler.source>17</maven.compiler.source>
    <maven.compiler.target>17</maven.compiler.target>
    <project.build.sourceEncoding>UTF-8</project.build.sourceEncoding>
  </properties>

  <!-- Dependencies -->
  <dependencies>
    <dependency>
      <groupId>org.springframework.boot</groupId>
      <artifactId>spring-boot-starter-web</artifactId>
      <version>3.1.0</version>
    </dependency>
    <dependency>
      <groupId>junit</groupId>
      <artifactId>junit</artifactId>
      <version>4.13.2</version>
      <scope>test</scope>
    </dependency>
  </dependencies>

  <!-- Build configuration -->
  <build>
    <plugins>
      <plugin>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-maven-plugin</artifactId>
      </plugin>
    </plugins>
  </build>

</project>
```

## Standard Directory Structure

Maven expects a specific directory layout. Following this convention means you never need to tell Maven where your files are.

```
my-app/
├── pom.xml
└── src/
    ├── main/
    │   ├── java/
    │   │   └── com/example/
    │   │       └── App.java          ← application source code
    │   └── resources/
    │       └── application.properties ← config files, templates
    └── test/
        ├── java/
        │   └── com/example/
        │       └── AppTest.java      ← test source code
        └── resources/
            └── test.properties       ← test config files
```

**Key directories:**

| Path | Purpose |
|---|---|
| `src/main/java` | Application source code |
| `src/main/resources` | Config files, templates, static assets |
| `src/test/java` | Test source code |
| `src/test/resources` | Config files used only during tests |
| `target/` | Generated by Maven — compiled classes, JARs (do not commit) |

## packaging

The `<packaging>` element tells Maven what output format to produce:

```xml
<packaging>jar</packaging>   <!-- default, produces a .jar file -->
<packaging>war</packaging>   <!-- web archive for servlet containers -->
<packaging>pom</packaging>   <!-- parent or aggregator module, no artifact -->
```

## properties

Properties let you define reusable values and reference them with `${property.name}`:

```xml
<properties>
  <spring.version>3.1.0</spring.version>
  <java.version>17</java.version>
</properties>

<dependencies>
  <dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-web</artifactId>
    <version>${spring.version}</version>
  </dependency>
  <dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-data-jpa</artifactId>
    <version>${spring.version}</version>  <!-- same version, one place to change -->
  </dependency>
</properties>
```

Changing the version in one place updates all references throughout the POM.
